# 0th Memory v2

**Date:** 2026-05-10
**Status:** active
**Durable:** yes — this fixes how 0th agents remember work across sessions, when they write knowledge, and how they keep repo-derived memory fresh; the chosen capture and freshness rules would be surprising without the research context.

## Decision

Evolve the KB into a workflow-integrated agent memory system rather than replacing it with a notes app or hosted memory service. Markdown decisions, KB pages, incident logs, and `CONTEXT.md` remain the auditable evidence layer; generated agent briefs, event-driven memory capture, read-set reconciliation, and repo sync metadata become the primary machine-facing memory layer.

Memory capture happens at meaningful workflow events, not at session end: decisions, verified root causes, user corrections, stale assumptions, slice completion, and repo updates. Durable claims are written through a canonical schema-validating writer, not by hand-editing JSONL. Normal exploration reconciles only the files and symbols the agent actually read against related memory; successful fast-forward pulls trigger a lightweight memory sync that marks affected claims as `needs_review` instead of rewriting the whole KB.

## Constraints

- Session end is unreliable because users can abandon threads, agents can crash, and context can compact; durable memory cannot depend on a final hook.
- Existing markdown remains valuable for provenance and human audit, but agents need brief/search/sync paths that do not require browsing raw notes by hand.
- Repo freshness must be conservative: fetch freely, pull only clean fast-forward branches, never reset or merge automatically.
- Memory maintenance needs lifecycle metadata: source paths, evidence, created/last-confirmed dates, scope, confidence, and supersession state.

## Not Doing

- No wholesale adoption of GBrain, MemPalace, agentmemory, Hindsight, or Supermemory before a local eval against real 0th recall questions.
- No whole-KB comparison on every code exploration; use read-set reconciliation keyed by source paths and symbols.
- No session-end-only capture; end hooks may summarize, but write-through event capture is the source of truth.
- No silent auto-compaction; reflection produces a report and marks stale/superseded memory explicitly.
